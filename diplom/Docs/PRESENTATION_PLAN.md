### 4. КЛЮЧЕВЫЕ КОМПОНЕНТЫ (2-3 минуты)

**4.1 Telegram-бот (HwsmTelegramBotSimple.java)**
- Точка входа для обработки сообщений
- Реализует интерфейс LongPollingUpdateConsumer
- Обрабатывает текстовые сообщения и callback-запросы
- Делегирует обработку специализированным обработчикам

**4.2 Обработчики (handlers)**
- **CommandHandler** - команды /start, /help, /stats
- **GameHandler** - логика игры, вопросы, ответы
- **CallbackHandler** - нажатия кнопок (подсказки, ответы)
- **KeyboardFactory** - создание inline-клавиатур

**4.3 Сервисы**
- **GameService** (538 строк) - управление игровым процессом
  - Методы: startGame(), answerQuestion(), useLifeline()
  - Логика несгораемых сумм
  - Сохранение/загрузка игры
  
- **QuestionService** (384 строки) - работа с вопросами
  - Метод: getRandomQuestionByDifficulty()
  - **Перемешивание ответов** - важный технический момент!
  - Кэширование перемешанных опций (ConcurrentHashMap)

**4.4 Сущности (Entity)**
- **Game** - текущая игра (статус, вопрос, призы)
- **User** - пользователь Telegram
- **Question** - вопрос с вариантами
- **QuestionOption** - варианты ответов
- **ShuffledQuestionOption** - перемешанные варианты для конкретной игры

---

### 5. ОСОБЕННОСТИ РЕАЛИЗАЦИИ (1 минута)

**Ключевые технические решения:**

1. **🔀 Перемешивание ответов (Shuffle)**
   - **Проблема:** в базе данных правильный ответ всегда в позиции A
   - **Решение:** ShuffledQuestionOption + кэш в памяти
   - Каждая игра получает уникальный порядок ответов

2. **💾 Сохранение прогресса**
   - Все ответы сохраняются в GameAnswer
   - Игра может быть продолжена после перезапуска
   - Состояние подсказок сохраняется в GameLifeline

3. **🏗️ Архитектура Telegram Bot API**
   - Использование SpringLongPollingBot
   - Реализация LongPollingUpdateConsumer
   - Внедрение зависимостей через конструктор (Dependency Injection)

## 🎯 Возможные вопросы и ответы

**Q: Почему выбрали именно эту архитектуру?**
A: Многослойная архитектура обеспечивает разделение ответственности, упрощает тестирование и поддержку.

**Q: Как решили проблему с перемешиванием ответов?**
A: Создали отдельную сущность ShuffledQuestionOption и кэш в памяти для каждой игры.

**Q: Что будете улучшать в проекте?**
A: Добавить больше вопросов, реализовать турнирный режим, улучшить UI.

**Q: Как обеспечиваете безопасность?**
A: Валидация входных данных, использование prepared statements, ограничение доступа к БД.

**Q: Какие были основные сложности?**
A: Перемешивание ответов, сохранение состояния игры, интеграция с Telegram API.
