# Исправление проблемы множественного перемешивания вариантов ответов

## Проблема
В QuestionService существовала проблема множественного перемешивания вариантов ответов для одного и того же вопроса:

1. **Корень проблемы**: Метод `getShuffledQuestionOptions` вызывался в разных частях системы для одного вопроса, и каждый раз происходило новое перемешивание
2. **Последствия**: Правильный ответ менялся между вызовами, что приводило к путанице пользователя
3. **Места вызова**: 
   - `KeyboardFactory.createQuestionKeyboard()` - для создания кнопок
   - `GameHandler.formatQuestionOptions()` - для отображения текста вопроса
   - `GameService.getSelectedOption()` - для проверки ответа
   - `GameService.getRandomWrongAnswer()` - для подсказки 50/50

## Решение

### 1. Добавлено кэширование перемешанных опций
```java
// Кэш для хранения перемешанных опций вопросов (questionId -> перемешанные опции)
private final Map<Long, List<QuestionOption>> shuffledOptionsCache = new HashMap<>();
```

### 2. Модифицирован метод getShuffledQuestionOptions
- Добавлена проверка кэша перед перемешиванием
- Перемешивание происходит только один раз за сессию
- Результат кэшируется для повторного использования

### 3. Добавлена очистка кэша
- `clearShuffledOptionsCache(Long questionId)` - очистка кэша для конкретного вопроса
- `clearAllShuffledOptionsCache()` - очистка всего кэша
- Автоматическая очистка при получении нового вопроса

### 4. Добавлены методы очистки кэша
- При каждом вызове `getRandomQuestionByDifficulty` кэш автоматически очищается
- Это гарантирует, что новые вопросы всегда получают свежее перемешивание

## Преимущества решения

1. **Консистентность**: Все части системы видят одинаковое перемешивание для одного вопроса
2. **Производительность**: Перемешивание происходит только один раз, а не при каждом вызове
3. **Безопасность**: Кэш автоматически очищается при переходе к новому вопросу
4. **Потокобезопасность**: Кэш обеспечивает одинаковые результаты в многопоточной среде

## Тесты

Создан тестовый класс `QuestionServiceShuffleTest` с проверками:
1. **testShuffledOptionsConsistency** - проверяет, что повторные вызовы возвращают одинаковый результат
2. **testShuffledOptionsThreadSafety** - проверяет консистентность в многопоточной среде
3. **testCacheClearedOnNewQuestion** - проверяет очистку кэша при получении нового вопроса

## Ожидаемый результат

После исправления:
- Перемешивание опций происходит только один раз за вопрос
- Все части системы видят одинаковые перемешанные опции
- Правильный ответ всегда соответствует перемешанным кнопкам
- Исчезла путаница с множественным перемешиванием
